# 게시판 구조 분석 가이드

## 현재 추가된 디버깅 기능

게시판 크롤링 시 다음 정보가 자동으로 수집됩니다:

1. **HTML 파일 저장**: `debug_board_page.html` 파일에 게시판 페이지 전체 HTML 저장
2. **구조 분석 로그**: 콘솔에 다음 정보 출력
   - 발견된 테이블 수
   - 발견된 tr 태그 수
   - 체크박스가 있는 tr 수
   - a.nr10 태그 수
   - post_view 링크 수
   - 첫 번째 게시글 행 샘플

## 추가로 필요한 정보 (선택사항)

게시판 구조를 더 정확히 파악하기 위해 다음 정보를 제공해주시면 도움이 됩니다:

### 1. 게시판 페이지 HTML 샘플
- `debug_board_page.html` 파일의 일부 (게시글 목록 부분)
- 또는 브라우저 개발자 도구에서 게시글 목록 부분의 HTML

### 2. 게시글 상세 페이지 구조 (선택사항)
- 게시글 제목 클릭 시 이동하는 상세 페이지의 HTML 구조
- 게시글 본문이 어떤 태그/클래스에 있는지

### 3. 특수한 구조 정보
- 게시글이 여러 페이지에 나뉘어 있는지 (페이지네이션)
- 동적 로딩이 있는지 (JavaScript로 나중에 로드되는지)
- 특별한 클래스명이나 ID 패턴

## 현재 파싱 로직

현재 코드는 다음 구조를 가정합니다:
- 게시글은 `<tr>` 태그 안에 있음
- 체크박스: `<input type="checkbox" name="idx">`
- 제목 링크: `<a class="nr10">` 또는 `href`에 `post_view` 포함
- 작성자/날짜: 같은 행의 `<td>` 태그 안에 있음

## 테스트 방법

1. 챗봇에 "에스티플리머스" 관련 질문 입력
2. 콘솔 로그 확인
3. `debug_board_page.html` 파일 확인
4. 로그에서 "게시판 구조 분석" 섹션 확인

## 문제 해결

만약 여전히 게시글을 찾지 못한다면:
1. `debug_board_page.html` 파일을 열어서 실제 HTML 구조 확인
2. 게시글 목록 부분의 HTML을 복사해서 제공
3. 콘솔 로그의 "구조 분석" 결과 확인
